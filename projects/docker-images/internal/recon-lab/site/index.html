recon-lab 0.0.4

Legal / ethical use only. You may point this at any http or https URL, but you are responsible for complying with law and obtaining authorization. Scan Info is passive recon (headers, HTML surface, common paths). Scan Vulnerability runs additional safe probes (TLS, CORS probe, TRACE, cookie flags, sensitive paths)—not a full exploit or CVE engine.

Any public or private URL is allowed. For TLS errors on self-signed hosts, set RECON_TLS_INSECURE=1 on the container.

From inside Docker: http://127.0.0.1:8092/ hits the container, not your host — use http://host.docker.internal:8092/ for a host-published port, or http://default-html:8080/ if this stack shares the app network (attach_networks).
