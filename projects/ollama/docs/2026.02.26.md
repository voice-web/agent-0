## Scripts added

| Script | Purpose |
|--------|--------|
| **scripts/ollama-bg** | Start Ollama in the background (`nohup ollama serve > ollama.log 2>&1 &`). Sets `OLLAMA_LLM_LIBRARY=cpu_avx2` so Intel Macs don’t get the MLX warning; comment notes the same var is in `~/.zshrc` for CLI commands (`ollama list`, `ollama ps`). |
| **scripts/ollama-kill** | Stop all Ollama processes (`pkill ollama`). |
| **scripts/system-resources** | Lightweight Python CLI for system resource info (macOS): `show info` (max CPU cores, memory, **Load [1m] [5m] [15m]**, process count), `show used` (current memory/CPU %), `show top [N] cpu\|memory` (top processes with % of max and % of used), `show pid <PID>`. Interactive REPL with readline, history, tab-complete. **Help:** `help show info\|used\|top\|pid` explains each subcommand (e.g. what Load and Processes mean). |
| **scripts/usage-monitor** | One-shot monitor: checks memory and CPU (same lightweight sources as system-resources); if above thresholds, **publishes** via macOS notification or log. Designed to be run by launchd every N seconds (e.g. 60). Config via env: `USAGE_MONITOR_MEM_PCT`, `USAGE_MONITOR_CPU_PCT`, `USAGE_MONITOR_PUBLISH` (notify\|log). |
| **scripts/com.local.usage-monitor.plist** | LaunchAgent plist to run `usage-monitor` every 60s; install by copying to `~/Library/LaunchAgents/` with script path substituted, then `launchctl load`. |

## Topics covered in conversation (reflected in docs)

- Running Ollama in the background and API-only usage (no need for `ollama run` for the API).
- Models load on first use; one server, one in-memory copy per model; multiple browser sessions use the same server/model.
- LLMs have no built-in file or OS access; tool use (function calling) is the way to let them “list files” or “run git” by executing tools in your app and passing results back.
- Colima: no UI, CLI only; images persist across `colima start`/`stop`; only `colima delete` removes them; offline build/run when base image and steps are local; disk space management and increasing VM disk.
- **system-resources** CLI (moved to agent-0): show info/used/top/pid; Load displayed as [1m] [5m] [15m]; `help show info|used|top|pid` for per-command explanations.
- **usage-monitor**: LaunchAgent that runs a one-shot check every 60s and notifies (or logs) when memory/CPU exceed thresholds; launchd config changes require unload then load to take effect.
